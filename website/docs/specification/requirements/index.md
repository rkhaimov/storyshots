# Требования

Исходя из указанных проблем можно сформулировать основные требования который обязан выполнять каждый тестовый сценарий.

:::tip
Напротив каждого требования указывается оценка в виде того, какие метрики оно удовлетворяет, а какие снижает. Каждой
метрике присваивается своё обозначение:

* 🛡️ Защита от регресса
* 🔧 Сопротивляемость рефакторингу
* 📈 Поддерживаемость
* ⚡ Быстродействие
  :::

Начать стоит с раздела [верификации поведения](/specification/requirements/assertion).
