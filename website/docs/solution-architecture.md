# Архитектура решения

## Основные компоненты

- **Превью-зона приложения**: Набор *историй*, описывающих различные сценарии использования приложения.
- **История**: Сценарий, в котором определяются действия над приложением и передаются специально подготовленные данные.
- **Менеджер**: Специальный компонент, управляющий исполнением историй и определяющий параметры среды выполнения (
  например, настройки браузера) и многое другое.

Интеграция `storyshots` в процесс разработки позволяет автоматизировать тестирование пользовательского интерфейса,
обеспечивая стабильность и предотвращая регрессии.
